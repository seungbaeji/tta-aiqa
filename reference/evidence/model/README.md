# Canonical Model Evidence

This directory contains prepared evidence from the one-shot sealed test evaluation.
`canonical-benchmark.json` is generated by `scripts/build_model_evidence.py` from
ignored runtime artifacts. It records the release decisions and whether deployment
may proceed.

`development-benchmark.json` preserves the validation-only benchmark used by the
freeze manifest. `model-bootstrap.json` records three model bundle hashes and MLflow
run IDs; only the baseline entry may have `deployed: true`.

The current result is a scenario review, not a model approval. Candidate B satisfies
the operating-point recall, uncertainty, precision, and false-negative checks, but
its test PR-AUC is below the baseline PR-AUC. The release policy therefore holds both
candidates. No post-test tuning or Candidate B deployment is allowed from this result.
